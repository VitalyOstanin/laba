laba

One desktop tray dashboard and CLI over a pluggable backend layer: connect several issue-tracker servers at once and see all their issues, pull requests and notifications in one place. GitHub is the primary backend today, OpenProject is supported too, and more trackers (Jira, YouTrack, GitLab, …) are planned — not yet implemented, but they fit the same design. Built with Tauri and a shared Rust core.

What it does

Pluggable backends

A backend is an issue tracker laba connects to. Add several servers of different trackers at once — a GitHub account and an OpenProject instance side by side — and work with all of them in one dashboard. GitHub and OpenProject are implemented today; more trackers (Jira, YouTrack, GitLab, …) are planned — not yet built, but they fit the same design.

GitHub in your tray

Your GitHub issues, pull requests and notifications in a small tray application for Linux, Windows and macOS. Uses the gh CLI, so no extra token setup.

Scriptable CLI

A laba command-line binary with JSON output by default, for automation and scripting — GitHub is the default backend.

Open source

MIT-licensed, a single Rust workspace. The desktop app talks to each backend directly through the core library.

Get started

Command-line interface (GitHub is the default backend):

laba server add gh github.com
laba wp list

The desktop app is distributed as release builds (including a Debian package) on the releases page. Build instructions are in the README.
